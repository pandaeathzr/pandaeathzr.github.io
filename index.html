var

var是ES6前的东西，教程网站称之为旧时的“var”，这样写道：“var 却是一头源自旧时代的怪兽。在现代脚本中一般不再使用，但它仍存在于陈旧的脚本里。”，因此，我基本上不会用到这个关键字。

但是心中仍存在着这样的疑问，它为何而淘汰？

提升(Hoisting)

var所声明变量的作用域要么是全局作用域，要么是函数作用域，这与我们之前所学的C或者C++里面的声明是有所区别的。无论在哪里被声明，其都会被当成在当前作用域顶部声明的变量。我们看下面几组代码：

inlineName变量在something()函数里面定义，显然在外面是无法访问得到的，所以报错ReferenceError: inlineName is not defined
—ReferenceError（引用错误） 对象代表当一个不存在的变量被引用时发生的错误。

当flag为true时，执行if语句分支，其定义了一个变量firstName；当为flase时，其执行console语句，打印firstName。在此会存在这样的以为，firstName变量在该分支内未曾定义，一旦运行，肯定会报一个ReferenceError。但是实际上，它打印的是undefined。
这样归结于变量提升(Hoisting)。我们上面说到，var所声明变量实际上都会被当成在当前作用域顶部声明的变量。不论在something()函数中那个位置声明，它实际上都会被提升到函数顶部，如下所示：

赋值操作在原定位置不变，而声明语句却是被提升到了顶部，此时它的值为undefined。

这导致了其作用域将覆盖整个函数，在一些情况下会存在出人意料的BUG。

作用域BUG

尽管我们在for循环中声明了i，但是可想而知，i的作用域是整个something()函数，因为，在setTimeout()回调函数执行时，所打印的i均是10，而不是我们所期待的由1递增到10。为此，ES6提出了新的声明关键字let && const。

const

const的特性与上面的let一样，但是它是一个常量。常量便是不可改变的，当我们用const声明并赋值基本类型（number,string,boolean,null,undefined,symbol）时，它们便不可再改变，一旦改变便会报类型错误。同时，声明变量的同时就要给变量赋值，否则亦会报错：

然而，当我们声明并赋值一个对象的时候，常量所存的为该对象的引用地址，常量的引用地址不可变，但是对象的值是可以改变的。

在使用for-in或者for-of的时候，使用const的行为与let一致，这是因为在迭代中，我们不是修改已有的绑定，而是创建一个新的绑定。

其他

当我们使用var在全局作用域声明变量时，它会再创建一个新的变量作为全局对象的属性（window对象），而let和const不会。

我不知道的Array

2019-10-23

本篇博客是对深入理解ES中第十章改进数组功能的部分学习

Array.of

在ES6前，我们要生成一个数组的话有两种方法：

  1. 数组字面量
  2. New Array()

第二种方法存在着一些问题，我们称之为怪异行为：

  • 传入一个数字类型的参数a，那么array.length = a
  • 传入一个非数字类型的参数a，那么array[0] = a
  • 传入多个参数，那么组成一个数组

所以为了规避这种不清不楚的怪异行为，ES6提出了Array.of()，其总会创建一个包含所有参数的数组。

PS: Array.of()不通过Symbol.species属性确定返回值类型，而是用of()中this的值来确定返回的正确类型。 这句话关系到Symbol这一新类型，之后在探讨

Array.from

在ES6前，我们想把一个类数组对象（例如arguments）转换成一个数组，我们可以采用以下的方法：

  • 迭代对象
  • Array.prototype.slice.call(arrayLike)

对于slice而言，它只需要根据length属性和数字型索引即可工作，因此第二种方法就可以把类数组对象复制成数组对象，但是这样有一个问题存在，语义不清楚，我们可能看不出他在干嘛，为此，ES6提出了Array.from()，其可以将传入的可迭代对象或类数组对象转化成数组，并且提供了一个类似map的映射函数(arguments, map, this in map) => {}

ES6新方法

Array.find/findIndex

在ES6前，数组搜索主要有两种方法：

  • indexOf()
  • lastIndexOf()

但是存在着局限性，每次只能搜索一个值。为此，ES6提供了条件搜索的方法：

  • find()
  • findIndex() 传入参数为两个，一个回调函数(value, index, array) => {},一个回调函数的this绑定

Array.fill

传入参数为三个，指定值，开始索引，结束不包含索引。因为是填充，显然会覆盖原数组。

如果索引为负值，那它会自动加上length

Array.copyWithin

传入参数为三个，开始填充索引，开始复制索引，结束不包含索引。因为是填充，显然会覆盖原数组。

掘金学习_前端程序员必须掌握之三角函数在前端动画中的应用

2019-10-07

本篇博客是对前端程序员必须掌握之三角函数在前端动画中的应用的学习

我要知道的知识点

window.requestAnimationFrame(callback)

告诉浏览器我希望执行一个动画，并且要求浏览器在下次重绘之前调用指定的回调函数更新动画。当然这仅仅只是画一帧，若我想在浏览器下次重绘之前继续更新下一帧动画，那么回调函数自身必须再次调用window.requestAnimationFrame()

requestAnimationFrame最大的优势是由系统来决定回调函数的执行时机。具体一点讲，如果屏幕刷新率是60Hz,那么回调函数就每16.7ms被执行一次，如果刷新率是75Hz，那么这个时间间隔就变成了1000/75=13.3ms，换句话说就是，requestAnimationFrame的步伐跟着系统的刷新步伐走。它能保证回调函数在屏幕每一次的刷新间隔中只被执行一次，这样就不会引起丢帧现象，也不会导致动画出现卡顿的问题。

而相比较，setTimeout()可能会由于不同的屏幕刷新率导致掉帧。

可以用下面的代码实现：

ctx.clearRect(x, y, width, height)

以点(x, y)为起点范围是(width, height)所有像素变成透明，并擦除之前绘制的所有内容的方法

CanvasRenderingContext2D.save()/.restore()

将当前状态放入栈中，保存 canvas 全部状态的方法/在绘图状态栈中弹出顶端的状态，将 canvas 恢复到最近的保存状态的方法

放一段绘制身体的代码就可以知道如何操作了

关于parseInt

parseInt(string, radix) 函数解析一个字符串参数，并返回一个指定基数的整数 当radix = undefined || 0 时,采用默认的行为：

  • 如果字符串 string 以"0x"或者"0X"开头, 则基数是16 (16进制).
  • 如果字符串 string 以"0"开头, 基数是8（八进制）或者10（十进制），那么具体是哪个基数由实现环境决定。Node使用的基数是10。因此，永远都要明确给出radix参数的值。
  • 如果字符串 string 以其它任何值开头，则基数是10 (十进制)。

当 radix = 1 时，似乎输出的都是 NaN。

当 传进来的参数 不是radix进制时，返回的也是 NaN

关于map

map(function(currentValue,index,arr), thisValue)方法创建一个新数组，其结果是该数组中的每个元素都调用一个提供的函数后返回的结果

什么是深拷贝，什么是浅拷贝在这里就不谈了，因为我已经知道了！

浅拷贝的实现方式：

  • Object.assign()
  • 展开符运算 ...
  • Array.prototype.slice()

Object.assign(target, ...sources)

方法用于将所有可枚举属性的值从一个或多个源对象复制到目标对象。它将返回目标对象

继承属性和不可枚举属性是不能拷贝的

原始类型会被包装为对象

这是在原始类型有默认迭代器的情况下，例如 String

深拷贝的实现方式：

  • JSON.parse(JSON.stringify(Object))
  • JQuery.extend()
  • lodash.cloneDeep

JSON.parse(JSON.stringify(Object))缺陷

  • 会忽略 undefined
  • 会忽略 symbol
  • 不能序列化函数
  • 不能解决循环引用的对象
  • 不能处理正则

博文中介绍说处理Date()时会转换错误，但是经过我的测试是不会的。
